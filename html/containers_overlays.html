Overlays

Larger, focused transient container used for gathering user input or interactions that does not navigate the user away from their current location. Overlays are used instead of a modal when more complex interactions are needed.

Guidelines

  • Overlays should be used when a single or multiple closing actions are needed. Example: just a cancel or save, cancel and third action
  • L3 card should be used to contain content
  • A 40% opacity Cool Gray (#52555A)
    page mask should cover the entire browser below the card for focus
  • Overlay should expand to fill the majority of the width or the page with a 24px margin around the entire browser view so mask can be seen. Height and width should always be fixed
  • Overlays should always have a header with no footer
  • CTA actions should be placed in the header on the right with the cancel button on the far right
  • There should not be horizontal scrolling inside of the overlay. If using a table, please use an inset table not a full card table
  • Clicking off the overlay onto the mask should not close the overlay, overlays should be closed using the cancel or another clear action

Sizing

  • Header has height of 64px
  • Header text is 24pt regular Cool Gray (#52555A)

  • Do not user a modal footer with an overlay
