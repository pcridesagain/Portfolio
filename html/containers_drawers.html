Drawers

Small transient view for gathering user information that does not force user focus, like a modal. Can be used to gather information both in context and out of context. The user can still interact with page views below drawer or with the drawer itself.

There are two themes of drawers, light and dark. Select the proper drawer based on visual hierarchy and focus that is needed.

Light Theme

The light theme was designed not to break the visual hierarchy of the page and blend in more within the content. Use this drawer when you need less user focus or when the user might need to click to other views in Canopy.

Dark Theme

The dark theme was designed to provide more focus without forcing the user to take direct action on the element (use modal or overlay for this).

Guidelines

  • Drawers can be triggered by the user or by the system. If the system needs to gather user input, the drawer should be used instead of a modal
  • A Drawer is made from an L3 card with no mask that slides in from the right side of the screen when triggered. It’s resting state is “offstage”
  • Max visual height of a drawer would be the screen height, you should always see the bottom of the card

Content

  • Headers should be used in all cases. Drawer headers do not have a bottom border, just a title and cancel “X”
  • Cancel “X” should always be present in the top right corner
  • A footer should always be used. There is no top border on footer. CTA is located in the bottom left. Modal CTA rules apply

Sizing

  • Width: 378px fixed
  • Padding: 24px
  • Border radius: 8px corner radius
  • Top margin: 16px (below the line of the primary navigation)
  • Right margin: 40px
  • Header text: 18px, 600 weight, Cool gray (#52555A)
  • DT Header: Bumble (#FFF)

Best Practices

  • When the system needs to gather user data out of context use drawer instead of modal because it’s less disruptive to the user. For instance, when IRS e-services login expires or calendar login expires
  • Drawers only slide "on-stage" from the right of the browser, never the bottom, top, or left side
