Buttons

Purpose/Usage

Buttons make common actions immediately visible and easy to perform with one click or tap.

Types

Primary Button

Primary buttons should be the single most important action on the page.

Secondary Button

Secondary buttons should be used for common actions when a primary button is present or can be used for a primary action when less emphasis is needed in the view.

Tertiary Button

A tertiary button can be used for multiple actions in a view when primary and secondary actions are already taken or when an action is needed that has low emphasis.

Flat Button

Flat buttons can be used as an alternative to primary or secondary buttons when needed for design aesthetic purposes or when primary and secondary buttons are both present.

Icon Button

An icon button is used when no text is needed for the action.

Global Add Button

Used to add content to Canopy out of context to minimize distraction.

Guidelines

Placement

  • In a page view, overlay or preview, buttons should be placed in the upper right hand corner.
  • In a modal or drawer buttons should be place in the bottom left hand corner.
  • In cards, buttons can be placed upper right or lower left depending on the use case.
  • Buttons should always be placed on cards, never floating on the canvas.

Content

The content of the buttons should:
  • Be human and relatable
  • Be clear and concise
  • Convey the actions expected ('Add' or 'Add Contact' = GOOD 'New'' = BAD)
  • Be written in "Title Case", except for "cancel" buttons
  • Not have labels written in ALL CAPS
  • Have labels with 3 words or less

Best Practices

  • For "cancel" action within modals, flat buttons are typically used.
  • Icons buttons are generally used for mensu and editing actions.
  • Icon button combos: no margin, 16px margin, 24px margin are all acceptable
  • Never use links for actions, links are used for navigation. Use flat or tertiasy buttons if you feel the urge to use a link for an action.
